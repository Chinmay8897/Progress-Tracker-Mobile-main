System & Objective Build a responsive, mobile-first Progressive App (PWA) called TaskCommand. TaskCommand is an intelligent team task management platform designed for centralized command. The application uses a modern Next.js and Tailwind CSS stack with a MongoDB backend structure. Make sure you remember that this is a mobile-app and not a website. Authentication & Authorization The system must support a single Head Manager account with full administrative privileges. Team Members must authenticate via email and password. Implement Role-Based Access Control (RBAC) to restrict member views to their permitted task scope. Define roles including Admin-Lite, Project Lead, Developer, and Support Agent. Data Models Create a Task model with the following required fields: Title, Description, Assignee, Due Date, and Priority Level. Include optional fields for Tags and Notes. The Task status must default to Open, and can transition to In Progress, Blocked, Done, or Cancelled. UI/UX Design System Ensure clarity by prioritizing task urgency and status without visual clutter. Maintain a minimum text contrast ratio of 4.5:1. Ensure all interactive touch targets are at least 44x44px. Implement strict visual priority coding: Critical is Red (#C0392B), High is Amber (#D4AC0D), Medium is Blue (#2E75B6), and Low is Gray (#7F8C8D). Core Screens to Generate 1. Head Manager Dashboard: Display a full task board with summary statistics at the top. Include filter toggles for Priority, Status, Assignee, and Due Date. Place a persistent, floating microphone button on the screen for voice commands. 2. Member Home: Display a filtered list showing only tasks assigned to the logged-in user. Sort the tasks by priority, followed by due date. Ensure no administrative controls are visible on this view. 3. Member Task Profile (Dashboard Drill-Down): Design a view that opens when the Head Manager clicks a member's name. Display a header with the member's avatar, name, and role badges. Include a horizontal summary strip showing counts for Open, In Progress, Blocked, and Done tasks. Render a scrollable list of tasks assigned strictly to this member. 4. Voice Command Panel (Overlay): Design a full-screen modal featuring an animated waveform. Include a live transcription text preview. Display a confirmation card to verify the parsed task intent before submission. 5. Task Detail View: Display full task information with a status update dropdown menu. Include an activity log and notification history panel. Development & Implementation Strategy To successfully guide the AI through the PRD requirements, build the application iteratively using the following phases: Phase 1: Database & RBAC Foundations Instruct the AI to scaffold the MongoDB schemas first. Ensure the schema logic strictly enforces that Team Members only query tasks where their user ID matches the assignee field, while the Head Manager bypasses this filter entirely. Phase 2: The Core UI Engine Focus on the styling parameters. Provide the AI with the exact hex codes and icon mappings for the Priority Visual Coding (Red, Amber, Blue, Gray) to ensure the visual hierarchy is established early. Phase 3: The Voice Interface Mockup Since web builders cannot immediately tie into complex NLP pipelines like OpenAI Whisper without explicit API keys, instruct the AI to build the frontend interaction of the Voice Command Panel first. Have it simulate a parsed string (e.g., extracting "Critical" and "Tomorrow") to populate the confirmation card. Phase 4: Notification Architecture The PRD outlines a complex priority-driven engine escalating reminders every 30 minutes to daily intervals. In a web environment, instruct the AI to set up standard Web Push APIs and service workers as the foundation for the Firebase Cloud Messaging requirement. Testing & Deployment Guidelines Role Simulation: You will need to create at least two permanent test accounts (one Head Manager, one Developer) to continuously verify that the progressive disclosure and scoped access UI elements are functioning correctly. State Management Testing: Thoroughly test the dashboard drill-down feature. Verify that when the Head Manager navigates to the Member Task Profile and presses back, the dashboard's scroll position and active global filters are perfectly preserved.
